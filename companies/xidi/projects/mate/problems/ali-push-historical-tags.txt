14:T3fbf,<h2>问题现象</h2>
<p>正式版 App 用户收到来自公司 App 的异常推送（乱码/测试文案），而用户安装的是<strong>正式版</strong>，理论上不应收到开发/测试环境的推送。</p>

<td>测试同学在新环境无法复现；问题用户有测试环境使用历史</td>

<h2>技术栈</h2>
<ul>
<li><strong>推送服务</strong>：阿里云移动推送（EMAS）</li>
<li><strong>客户端</strong>：React Native + <a href="https://www.npmjs.com/package/react-native-aliyun-emas"><code>react-native-aliyun-emas</code></a></li>
<li><strong>推送策略</strong>：基于**标签（Tag）**定向推送</li>
</ul>
<h2>阿里推送标签机制</h2>
<p><strong>推送流程：</strong></p>
<pre class="mermaid-source"><code class="language-mermaid">flowchart LR
    A[App 启动/登录] --&gt; B[bindTag 绑定标签]
    B --&gt; C[设备 DeviceId]
    C --&gt; D[(阿里云 Tag 存储)]
    E[服务端按 Tag 推送] --&gt; D
    D --&gt; F[命中设备收到推送]</code></pre>
<p><strong>关键规则（官方文档）：</strong></p>
<ul>
<li>标签与<strong>设备 ID</strong>绑定，不是与账号绑定</li>
<li>一个设备可绑定不限数量标签；一个标签可绑定不限数量设备</li>
<li>换设备后，旧设备上的标签不再生效</li>
<li>多设备同账号：各设备独立维护标签集合，按 Tag 推送时取<strong>并集</strong></li>
</ul>

<h3>2. 清理历史脏标签</h3>
<ul>
<li>利用阿里云 API <strong>查询当前设备已绑定的全部标签</strong></li>
<li>每次重新绑定时，<strong>先解绑所有已知/历史标签</strong>，再绑定当前环境应有的标签</li>
<li>对存量用户：发版后首次启动即完成一次全量清理</li>
</ul>
<h3>3. 避免 useEffect 异步任务堆积</h3>
<p><code>useEffect</code> 中依赖频繁变化时，会连续触发多次 bind/unbind，异步任务堆积导致<strong>执行顺序不可控</strong>。</p>
<p><strong>建议：</strong></p>
<ul>
<li>用 <code>destroyRef</code> / AbortController 串行化异步操作</li>
<li>或改用专门的状态机/队列管理标签生命周期</li>
<li>参考：<a href="https://overreacted.io/a-complete-guide-to-useeffect/">用 useEffect 优雅管理 React 中的异步操作</a></li>
</ul>
<p><strong>修复后的执行顺序：</strong></p>
<pre class="mermaid-source"><code class="language-mermaid">sequenceDiagram
    participant App
    participant Ref as destroyRef
    participant Ali as 阿里云推送

    App-&gt;&gt;Ref: 等待上次 unbind 完成
    Ref--&gt;&gt;App: finally
    App-&gt;&gt;Ali: bindTag（当前环境标签）
    Note over App,Ali: 依赖变更 / 组件卸载
    App-&gt;&gt;Ali: unbindTag（全部相关标签）
    Ali--&gt;&gt;Ref: 更新 destroyRef Promise</code></pre>
<h2>经验总结</h2>
<ol>
<li><strong>数据类 Bug 必须考虑历史数据</strong>——旧逻辑缺陷产生的脏数据，不会随代码修复自动消失</li>
<li><strong>标签/订阅类功能要有「全量清理 + 增量绑定」策略</strong>，不能只做增量 bind</li>
<li><strong>环境切换（测试 ↔ 正式）</strong> 是标签类推送的高危场景，需单独验证</li>
<li><strong>useEffect + 异步副作用</strong> 要注意依赖抖动和任务乱序，关键路径应串行化</li>
