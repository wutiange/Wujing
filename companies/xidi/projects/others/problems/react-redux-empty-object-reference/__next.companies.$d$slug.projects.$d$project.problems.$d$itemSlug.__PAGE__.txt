<h2>根因分析</h2>
<h3>useSelector 的比较机制</h3>
<p><code>useSelector</code> 对 selector <strong>返回值</strong>做<strong>引用相等</strong>（<code>===</code>）比较：</p>
<ul>
<li>返回值与上次相同 → 不触发重渲染</li>
<li>返回值不同 → 订阅组件重渲染</li>
</ul>
<pre class="mermaid-source"><code class="language-mermaid">flowchart TD
    A[userInfo 为 undefined] --&gt; B[&quot;selector 返回 {} （新对象）&quot;]
    B --&gt; C[useSelector 认为数据变化]
    C --&gt; D[组件重渲染]
    D --&gt; A</code></pre>
<p><strong>关键：<code>|| {}</code> 在每次 selector 执行时都会创建一个全新的空对象。</strong></p>
<h3>无限循环链路</h3>
<pre class="mermaid-source"><code class="language-mermaid">flowchart TD
    A[组件渲染] --&gt; B[&quot;useSelector 返回新 {}&quot;]
    B --&gt; C[&quot;useEffect([userInfo]) 触发&quot;]
    C --&gt; D[&quot;setNameArr([name]) 新数组&quot;]
    D --&gt; E[组件再次渲染]
    E --&gt; A</code></pre>
<table>
<thead>
<tr>
<th>步骤</th>
<th>说明</th>
</tr>
</thead>
<tbody>
<tr>
<td>1</td>
<td><code>userInfo</code> 为 <code>undefined</code>，selector 返回<strong>新</strong> <code>{}</code></td>
</tr>
<tr>
<td>2</td>
<td><code>useEffect</code> 依赖 <code>[userInfo]</code>，引用变了 → 执行 <code>setNameArr</code></td>
</tr>
<tr>
<td>3</td>
<td><code>setNameArr([userInfo.name])</code> 返回<strong>新数组</strong> → 触发渲染</td>
</tr>
<tr>
<td>4</td>
<td>重新渲染 → selector 再次返回<strong>新</strong> <code>{}</code> → 回到步骤 2</td>
</tr>
</tbody>
</table>
<h3>为何 <code>setNameArr('')</code> 不会死循环？</h3>
<p>去掉 <code>|| {}</code> 后，<code>userInfo</code> 在数据到达前稳定为 <code>undefined</code>，<code>useEffect</code> 依赖不再每次变化。此时 <code>setNameArr('')</code> 因<strong>值相等</strong>（<code>Object.is('', '')</code>）被 React 跳过，不会额外触发渲染：</p>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="tsx" data-theme="ayu-dark"><code data-language="tsx" data-theme="ayu-dark" style="display: grid;"><span data-line=""><span style="color:#FF8F40">const</span><span style="color:#BFBDB6"> userInfo </span><span style="color:#F29668">=</span><span style="color:#FFB454"> useAppSelector</span><span style="color:#BFBDB6">(</span><span style="color:#D2A6FF">state</span><span style="color:#FF8F40"> =></span><span style="color:#BFBDB6"> state</span><span style="color:#F29668">.</span><span style="color:#BFBDB6">app</span><span style="color:#F29668">.</span><span style="color:#BFBDB6">userInfo)</span><span style="color:#BFBDB6B3">;</span><span style="color:#5A6673;font-style:italic"> // 不再 || {}</span></span>

<h2>经验总结</h2>
<ol>
<li><strong>永远不要在 selector 里写 <code>|| {}</code> / <code>|| []</code></strong>——每次渲染产生新引用</li>
<li><code>useSelector</code> 使用<strong>引用相等</strong>比较，对象/数组默认值必须在组件外或 reducer 中定义</li>
<li><code>useEffect</code> 依赖项避免不稳定引用；优先依赖<strong>原始值字段</strong>（<code>userInfo?.name</code>）</li>
<li>Redux 初始 state 用 <code>null</code> 或稳定的空对象，避免 <code>undefined</code> + 兜底对象的组合</li>
<li>排查渲染次数：对比<strong>理论执行次数</strong>与 <code>console.log</code> 实际次数，差异大时先查引用稳定性</li>
</ol>
